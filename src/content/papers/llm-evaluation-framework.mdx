## 1. Introduction

As Large Language Models (LLMs) are increasingly integrated into production systems, the need for robust evaluation methodologies has become critical. Traditional metrics like BLEU or ROUGE are often insufficient for assessing the quality, safety, and helpfulness of open-ended LLM generations.

This paper outlines a pragmatic framework for evaluating LLM outputs in real-world applications.

## 2. Evaluation Framework

Our proposed framework consists of three tiers of evaluation:

1.  **Deterministic Heuristics**: Fast, cheap rules for checking constraints (e.g., regex matching, formatting, length).
2.  **LLM-as-a-Judge**: Utilizing a more capable LLM to evaluate the outputs based on specific rubrics (e.g., relevance, coherence, toxicity).
3.  **Human-in-the-Loop Validation**: Continuous sampling of outputs for human review to calibrate the automated evaluation methods.

### 2.1 The LLM-as-a-Judge Pipeline

We found that providing the evaluating LLM with a detailed rubric and few-shot examples significantly increased its correlation with human judgments.

```javascript
// Example of invoking an LLM judge
async function evaluateOutput(prompt, generatedText, rubric) {
  const evaluationPrompt = `
    Evaluate the following generated text based on the rubric.
    Prompt: ${prompt}
    Generated Text: ${generatedText}
    Rubric: ${rubric}
    
    Provide a score from 1 to 5 and a brief justification.
  `;
```

## 3. Findings

Implementing this framework in three varied production applications revealed several key insights:

*   Deterministic heuristics catch approximately 40% of critical failures (especially formatting errors).
*   LLM-as-a-judge correlates with human judgment at $\rho = 0.82$ when provided with a detailed rubric.
*   Continuous human calibration is necessary, as distribution shifts in user inputs can degrade the accuracy of automated evaluation over time.

## 4. Conclusion

A multi-tiered evaluation approach is essential for maintaining the reliability and safety of LLM applications in production. Further research is needed to reduce the latency and cost of LLM-as-a-judge methods.
