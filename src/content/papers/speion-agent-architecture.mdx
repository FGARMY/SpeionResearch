## 1. Introduction

The increasing complexity of modern software systems has driven interest in autonomous code generation. While large language models (LLMs) have shown remarkable capabilities in generating code snippets, their performance degrades on complex, multi-file tasks requiring architectural understanding.

This paper proposes a multi-agent architecture where distinct agents take on specialized roles (e.g., Planner, Coder, Reviewer) to collaboratively solve software engineering problems.

## 2. Methodology

We designed a system consisting of three primary agent types:

1.  **Planning Agent**: Responsible for breaking down high-level requirements into actionable tasks and creating an implementation plan.
2.  **Coding Agent**: Responsible for generating the source code for each task defined by the Planning Agent.
3.  **Reviewing Agent**: Responsible for reviewing the generated code, identifying bugs, and suggesting improvements.

Our evaluation dataset consisted of 50 complex software engineering tasks, ranging from implementing new features in existing codebases to resolving multi-file bugs.

### 2.1 Experimental Setup

We compared our multi-agent system against a single-agent baseline using the same underlying LLM (GPT-4o). Both systems were evaluated on code correctness, execution success, and architectural adherence.

## 3. Results

The multi-agent architecture significantly outperformed the single-agent baseline across all metrics. 

| Metric | Single-Agent | Multi-Agent | Improvement |
| :--- | :--- | :--- | :--- |
| Code Correctness | 65% | 88% | +35% |
| Execution Success | 52% | 81% | +55% |
| Architectural Adherence | 45% | 78% | +73% |

As shown in the table above, the most substantial improvement was observed in architectural adherence, suggesting that the specialized Planning Agent effectively guided the Coding Agent.

## 4. Discussion

The results indicate that mimicking human collaborative workflows through multi-agent architectures is a promising approach for autonomous software engineering. Future work will explore more dynamic agent interactions and larger teams of specialized agents.
